# Design progetto PII

Facciamo in C++

- CLI 
- File di configurazione YAML
	- Campo -> Grafo diretto (più estendibile) (grafo indiretto, griglia) (con formati particolari)
	- Agenti -> Numero, etichetta, Posizione iniziale e finale (goal)
	- File generale per il benchmark (campo, agenti, l'eseguibile dell'algoritmo) altri parametri [metriche, numero di tentativi].
		-> in futuro (vincoli particolari) Edge, Vertex
- Legge un output come formato di azioni
	- Muovi (elemento in posizione iniziale in posizione finale, istante di tempo in cui avviene l'azione, "label").
	- Lista azioni per agente (non è necessario l'istante di tempo, in ordine).
	- Controllare quanto è facile da implementare.
- Processa Output -> calcola le metriche, vede conflitti
- Export come CSV.
